# LINE Auto-Pilot（仮）要件定義書 Ver 3.1

## 1. 製品・サービス概要
*   **製品名**: LINE Auto-Pilot（仮）
*   **コンセプト**: 「専門知識不要、Webアプリ上の直感的な操作で、高度なLINE自動応答・予約・ポイントシステムを構築できるSaaSプラットフォーム」
*   **提供価値**:
    *   開発技術を持たない店舗オーナーが、自らDX（業務効率化）を推進できる環境。
    *   GASを一切使用しない堅牢なデータ管理と、モダンな操作感の提供。

## 2. システムアーキテクチャ
フロントエンド（Vercel）とバックエンド（Supabase）を完全に分離し、スケーラビリティと高速なレスポンスを両立。

| レイヤー | 採用技術 | 役割 |
| :--- | :--- | :--- |
| **フロントエンド** | React (Vite) / Tailwind CSS | 管理画面UI、設定フォーム、データ可視化（Recharts等） |
| **ホスティング** | Vercel | フロントエンド配信、GitHub連携によるCI/CD |
| **バックエンド/DB** | Supabase | 認証(Auth)、PostgreSQL、Realtime通信、ストレージ |
| **Webhook処理** | Supabase Edge Functions | LINEからの信号受信、OpenAI連携、ビジネスロジックの実行 |
| **外部連携** | LINE API / OpenAI API | Messaging API / LIFF、AI対話生成 |

## 3. データベース物理設計（Supabase/PostgreSQL）
マルチテナント（複数顧客管理）に対応し、データ整合性を重視。

*   **profiles**: 管理者（ユーザー）情報、プラン（free/pro）、契約ステータス。
*   **line_accounts**: Messaging APIキー、アクセストークン、bot_id（ユニークURL用）。
*   **auto_responses**: キーワード応答ルール（キーワード、返信文、AI優先フラグ）。
*   **reservations**: 予約データ（日時、LINEユーザーID、ステータス、店舗側メモ）。
*   **points**: 顧客別ポイント管理（現在のポイント数、最終更新日、ポイント履歴URL）。
*   **customer_logs**: LINEユーザーとの接触履歴、送信メッセージログ。

## 4. 機能詳細

### 4.1 Web管理画面（React UI）
GASによる管理を廃止し、すべての操作をWeb UI上で行う。

*   **統合ダッシュボード**: 友だち数、予約状況、ポイント発行推移のグラフ表示。
*   **CRM（顧客管理）**: スプレッドシートを介さず、ブラウザ上で直接顧客情報を検索・編集。
*   **応答設定エディタ**: リスト形式での応答ルール作成。プレビュー機能付き。
*   **予約・ポイント操作**: 管理者が手動で予約を追加・変更したり、ポイントを付与する機能。
*   **CSVエクスポート**: 解析用にデータをスプレッドシートやExcelで扱いたいユーザー向け。

### 4.2 LINEボット機能（Edge Functions）
*   **高速Webhook応答**: Edge Functions (Deno) による低遅延レスポンス。
*   **予約・ポイントLIFF**: LINE内で開く高機能な専用UI。予約カレンダーやマイページを提供。
*   **AI自動接客**: 既存の応答ルールに該当しない場合、OpenAIが店舗情報を基に回答。

## 5. セキュリティ・運用方針
*   **Row Level Security (RLS)**: SupabaseのRLSにより、**「店舗Aのデータは、店舗Aの管理者のみが読み書きできる」**ことをDBレベルで強制。
*   **API保護**: LINE Webhookの署名検証をEdge Functions内で厳格に実施。
*   **環境変数管理**: Vercel/Supabaseの環境変数機能を使い、認証キーを安全に保持。
